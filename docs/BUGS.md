| D-002 | Pipeline fabricates content when narrator stops mid-sentence. Two failure points: (1) Whisper (`gpt-4o-transcribe`) hallucinates completions on silence/abrupt endings — worsened by our recipe-vocabulary `initial_prompt` priming it. (2) Structure LLM prompt says "steps must be in cooking order even if narrated non-linearly" and "infer dish_name from context" — both invite the model to fill gaps that weren't narrated. Fix: added `temperature=0` to Whisper call, shortened `initial_prompt` to a terse vocab list via new `build_glossary_terms_list()` (omits meanings so the prompt doesn't suggest content), rewrote structure rules to "only include what was explicitly stated", removed reorder/infer language so steps/dish_name stay as-spoken or null with a `review_flags` note instead of being filled in. **2026-07-01 additional fix**: added `_strip_hallucination_loops()` post-processing to collapse consecutive repeated word/sentence runs (Whisper loops on silence — e.g. "Add sugar." × 500 or "మోటియింది" × 500). **2026-07-06 follow-up**: `_MAX_CONSECUTIVE_REPEATS` reduced from 2 → 1 after real-device testing showed the loop tail still appearing once ("పిండీ పిండీ." appearing twice). Recipe narration never legitimately repeats the identical sentence consecutively, so max 1 is correct. Real-model eval (`tests/evals/`) still recommended. | High | Fixed 2026-07-06 | `tools/transcribe.py` |
